# 模型档位评测 (eval)

用一份带"合理价"基准的数据集，跨三个档位（经济/标准/旗舰）跑雪板鉴定，量化对比
**品牌识别准确率 / 估价相对误差 / token / 成本 / 耗时**，用数据回答"哪档最划算"。

## 怎么用

1. **准备数据集**：编辑 `eval/cases.json`。初始已用仓库自带的 `examples/` 9 张图填了 3 个 case。
   - `images`：图片路径（相对仓库根目录），同一块板的多视图放一组。
   - `true_brand`：正确品牌（大写英文，如 `BURTON`）。
   - `fair_price`：**你认可的合理成交价**（人民币）。当前是占位值，请改成你判断的真实合理价（这是评测的"标尺"）。
   - 想更可信就多加几条 case（建议 10–20 块板，覆盖热门/战损/老款/冷门）。

2. **跑评测**（在仓库根目录）：
   ```bash
   python eval/run_eval.py
   ```
   需要本机有效的 API key（同跑 app 一样）。脚本会依次切三档、逐 case 调用。

3. **看结果**：
   - `eval/report.md`：对比表，可直接贴简历/README。
   - `eval/results.csv`：逐 case 明细（预测品牌、估价、误差、token、成本）。

## 说明

- 默认 `USE_HINT = False`，测模型"裸识别"能力；想测带提示效果，把 `run_eval.py` 顶部改 `True`。
- 成本是估算，按 `utils/usage.py` 里的 `PRICES` 价格表算——把真实单价填进去就准。
- 误差 = |估价中值 − 合理价| / 合理价，越低越贴近你的市场判断。

## 这张表能讲什么（面试）

> "我建了评测集，用真实合理价做基准，量化对比三档模型。结论是 **更贵的旗舰档并非误差最低**——它过度触发折旧规则；标准档在准确率和成本间最优。所以选型靠数据，不靠直觉。"
