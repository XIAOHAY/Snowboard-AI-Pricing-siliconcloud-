# 🏂 二手雪板 AI 估价 Agent (Snowboard AI Pricing Agent)

> 一个**多模态、工具调用（tool-calling）**的二手雪板估价智能体：上传雪板照片即可得到成色鉴定与确定性估价，并能就品牌保值、真实行情、保养知识进行多轮对话。
> 核心特色：**感知/决策分离治理大模型幻觉报价 · 模型档位（梯度收费）· 真实闲鱼行情交叉验证 · 自带评测集与用量观测。**

## ✨ 项目亮点

- **从固定流水线 → 自主 Agent**：用 `bind_tools` 让大模型按用户意图**自主决定**调用哪个工具（图像鉴定 / 品牌保值 / 真实行情 / 装备知识），单一对话入口处理估价与咨询。
- **多模态 + 感知/决策分离**：视觉模型（Qwen3-VL）只输出成色、损伤、品牌等结构化特征；**确定性规则引擎**据品牌保值梯队与折旧曲线算价，**治理大模型"幻觉报价"**、价格可复现、可解释。
- **模型档位（梯度收费）**：经济 / 标准 / 旗舰三档，用户在成本与质量间自选；模型 ID 集中配置、运行时热切换、换模型零代码改动。
- **真实行情交叉验证**：`search_market_price` 接入闲鱼（Goofish）真实在售挂牌价快照，给估价做市场对标（并明确标注挂牌价≠成交价）。
- **自带评测集 + 用量观测**：带真实合理价基准的评测脚本量化对比各档"准确率 / 估价误差 / token / 成本 / 延迟"；每次调用的模型与花费实时可见。

## 🧠 架构

```
用户一句话(可带图)
        │
        ▼
  Agent (LLM 决策) ── 按意图自主选工具 ──┐
        │                               │
        ├─ appraise_snowboard  →  Qwen3-VL 看图(感知) → 规则引擎算价(决策)
        ├─ search_market_price →  闲鱼行情快照(挂牌价交叉验证)
        ├─ get_brand_liquidity →  品牌保值梯队
        └─ query_gear_knowledge → 装备知识(保养/选板/术语)
        │
        ▼
   工具结果喂回模型 → 用"老炮"口吻整合输出（鉴定表/价格拆解/点评/建议）
```

**为什么是 Agent 而不是流水线**：旧版调用顺序写死，只能"传图→估价"；现在决策权交给模型，同一套代码能估价、答品牌、查行情、讲知识，加能力＝加一个工具。

**为什么估价不让大模型直接报价**：大模型报价有幻觉、不可复现。感知（看图）与决策（算价）分离后，价格由确定性公式给出，可解释、可校准。

## 📊 评测（数据驱动选型）

自建评测集（带真实合理价基准），跨三档量化对比：

| 档位 | 品牌识别准确率 | 估价平均相对误差 | 平均 token | 成本(估) | 平均耗时 |
|---|---|---|---|---|---|
| 经济档（VL-8B） | 33% | 60% | 9165 | ¥0 | 22s |
| **标准档（VL-32B）** | **67%** | **40%（最低）** | 9178 | ¥0.052 | 62s |
| 旗舰档（397B） | 67% | 74% | 18937 | ¥0.235 | 94s |

**结论（反直觉）**：最贵的旗舰档误差最高、成本约 5 倍——"模型越大越好"在本业务不成立；且三档误差都偏高，说明**瓶颈在定价规则标定而非模型规模**。据此选定标准档为默认，并把后续重心放在规则校准上。

> 复现：`python eval/run_eval.py`（产出 `eval/report.md` 与 `eval/results.csv`）。

## 🚀 快速开始

```bash
git clone <repo>
cd <repo>
pip install -r requirements.txt
# 配置 API Key：Streamlit Secrets 或 .env 里设 DASHSCOPE_API_KEY（硅基流动 key 即可）
```

两个入口：

```bash
streamlit run agent/app_agent.py      # ✅ Agent 版（对话 + 自主工具调用 + 档位选择，推荐）
streamlit run app_ui_deploy.py        # v1：表单式估价（多图融合 + 演示案例）
```

刷新行情数据（可选）：`pip install apify-client` + 设 `APIFY_TOKEN`，再 `python data/build_market_snapshot.py`。

## 🗂️ 目录结构

```
agent/        Agent 层：tools.py(工具) · snowboard_agent.py(决策循环) · app_agent.py(对话UI)
llm/          视觉与对话模型封装（Qwen-VL / 文本）
pricing/      确定性定价引擎 + 专家点评
utils/        多视图融合 · 用量/成本观测
data/         品牌原价表 · 装备知识库 · 真实行情快照 + 抓取脚本
eval/         评测集与评测脚本（准确率/误差/成本对比）
config.py     集中配置：API、Base URL、模型档位
```
